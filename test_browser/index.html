TinyData project

TinyData предназначен для работы с неструктурированными данными произвольной глубины вложения в любой JS-среде (проверена работа в node.js и в современных браузерах)

Источником таких данных может быть документ-ориентированнная DB, например MongoDB или любой иной источник JSON данных.

Установка и использование

        

           
          Для использования в браузере необходимо подключить на страницу lodash и TinyData в полной или минифицированной форме.
        

        
Для использования в Node.js просто используйте команду npm install tiny-data.

      

      

        
Решаемые инструментом задачи

        
TinyData  упрощает взаимодействие с вложенными данными, помогая осуществлять 
          поиск и агрегирование используя относительно простой интерфейс.
        

      

      

        
Технологии

        

           
          Фактически TinyData подготавливает (и кеширует) материализованные пути для переданного
          объекта, позволяя стоить над ними запросы как с обычными текстовыми данными.
        

        

          В самом простом варианте для формирования запроса применяются 
          регулярные выражения.
        

      

      

        
Например так может выглядеть запрос для получения рейтинга каждого комментария:

        
 /^(\d+\.comments\.\d+\.(?:comments\.\d+\.)*)rating\.([^.]+)/

      

      
 
        
фрагмент ответа...

        

          

        

      

      

        
 
             
            Посмотреть полный код агрегирования рейтинга можно в разделе Примеры или Демо.

      

      

        

           
          Кроме того, для поиска можно использовать строку, которая будет преобразована
          в регулярное выражение, или даже функцию.
        

        

          Функция является самым мощным вариантом, но она требует больших телодвижений
          и более глубокого понимания происходящего.
        

      

      

        

           
          Для целей "причесывания" результатов может применяться финализатор -
          так же функция, в которой можно работать как над значаниями, так и над ключами.
        

      

      

        
Тюнинг

        

           
          В целях ускорения работы над большими объемами данных реализован 
          механизм настройки процесса стрингификации (материализации пути) с фильтрацией, 
          кроме того значения в виде длинных строк (более 120 символов) усекаются
          до сообщения __LONG_TEXT__|355| где 355 - длина в символах (пользовательская настройка длины будет реализована, но позднее).
        

        

           
          Примеры тюнинга и его результативность можно посмотреть в разделе Тесты, пункт Do stress test on big data.   
        

        

           
          На данном этапе тюнингом можно добиться примерно 1.5-кратного прироста производительности,
          если же этого мало - вероятнее всего следует использовать нативные итераторы.
